JSON Format

Opportunity Tasks are represented as JSON objects which have the following keys:

{% include table-header-readonly.html %}
id {{ site.format.type.integer }} Automatically assigned when creating a task
name {{ site.format.type.string }} The task name.
notes {{ site.format.type.string}} The task notes.
status {{ site.format.type.string }} "done" if the task is completed, "todo" otherwise.
due_date {{ site.format.type.date }} Date on which this task is due, or null if the task has no due date.
done_date {{ site.format.type.date }} Date on which this task was completed, or null if the task has no done date.
time_spent {{ site.format.type.double }} The time spent to complete this task, or null if the task is not done yet.
assigned_to {{ site.format.type.object }} Employee to which this task is assigned, or null if the task is unassigned.
customer {{ site.format.type.object }} Compact representation of the customer linked to this task, or null if no customer linked.
contact {{ site.format.type.object }} Compact representation of the contact linked to this task, or null if no contact linked.
opportunity {{ site.format.type.object }} Compact representation of the opportunity linked to this task, or null if no opportunity linked.
