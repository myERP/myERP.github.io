h2. GET items/:id

p. Returns all the information about the item.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: GET
* Response Format: JSON

h3. Example Request

{% highlight bash %}
curl -X GET {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.items.url}}/:id {{site.api.auth_pattern}}
{% endhighlight %}

p. JSON response sample (Company with no Contacts)

{% highlight bash %}

  "id": 230071,
  "external_id": "NIN-WII",

span(anchor#item_put). ~

h2. PUT items/:id

p. Update a single item.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Non editable fields: id, type

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.items.url}}/:id {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T fieldsToUpdate.json
{% endhighlight %}

p. JSON response

The up-to-date JSON of the item you've just modfied.

h2.  GET items

p. Returns the list of all your items.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: GET
* Body Format: JSON
* Response Format: JSON

h3. Example Request

{% highlight bash %}
curl -X GET {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.items.url}} {{site.api.auth_pattern}}
{% endhighlight %}

p. JSON response

The list of all your items, in a JSON format.


span(anchor#items_post). ~

h2. POST items

p. Create a single item.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: POST
* Body Format: JSON
* Response Format: JSON
* Mandatory fields : type, name

h3. Example Request

{% highlight bash %}
curl -X POST {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.items.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T newItem.json
{% endhighlight %}

p. JSON response

The complete JSON of the item, as it was created by the system.


span(anchor#items_put). ~

h2. PUT items

p. Create and Update several items in a row.

h3. Resource Information

* Rate limited: Yes
* Batch limit: {{ site.api.batch_limit }}
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Mandatory fields for creation: type, name
* Non editable fields for edition: id, type

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.items.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T listOfItems.json
{% endhighlight %}

p. JSON response

A JSON list made of up-to-date info of items successfully edited or created, and error messages for unsuccessful ones.
