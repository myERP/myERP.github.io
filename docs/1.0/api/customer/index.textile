h2(blue). GET customers/:id

p. Returns all the information about the customer or lead.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: GET
* Response Format: JSON

h3. Example Request

{% highlight bash %}
curl -X GET {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}}/:id {{site.api.auth_pattern}}
{% endhighlight %}

span(anchor#customer_put). ~

h2(blue). PUT customers/:id

p. Update a single customer or lead.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Non editable fields: id, full_name

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}}/:id {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T fieldsToUpdate.json
{% endhighlight %}

p. JSON response

The up-to-date JSON of the customer you've just modfied.

span(anchor#customers_post). ~

h2(blue). POST customers

p. Create a single customer or lead.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: POST
* Body Format: JSON
* Response Format: JSON
* Mandatory fields : type, status, company_name (for Company)/last_name (for Individual)

h3. Example Request

{% highlight bash %}
curl -X POST {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T newCustomer.json
{% endhighlight %}

p. JSON response

The complete JSON of the customer, as it was created by the system.
