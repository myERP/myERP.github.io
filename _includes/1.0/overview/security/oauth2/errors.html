Handling Errors

When any error occurs during the OAuth2, the user will be redirect to the redirect_uri which was sent during the start of the flow.

Note
When the application starts the flow omitting or sending an invalid redirect_uri or client_id, a default error page will be displayed to the user given that the Authorization Server cannot thrust the redirect_uri.

OneUp Authorization Server will redirect the call and will send the following query parameters:

Parameter Description
error An unique string id indicating which error occurred
Your application can use this id to map possible errors and handle it
See the table bellow for possible values for this field
error_description A description of the error occurred
error_hint A possible hint to fix the error
state The state parameter value which was sent during the start of the flow

The same parameters listed above will be returned for requests made without user interaction (i.e: code exchange, refresh token) in a JSON format.
