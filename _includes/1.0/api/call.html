JSON Format

Calls are represented as JSON objects which have the following keys:

{% include table-header-readonly.html %}
id {{ site.format.type.integer }} Automatically assigned when creating a call
name {{ site.format.type.string }} The call name.
notes {{ site.format.type.string}} The call notes.
status {{ site.format.type.string }} "done" if the call is completed, "todo" otherwise.
qualification {{ site.format.type.string }} The call qualification.
due_date {{ site.format.type.date }} Date on which this call is due, or null if the call has no due date.
done_date {{ site.format.type.date }} Date on which this call was completed, or null if the call has no done date.
time_spent {{ site.format.type.double }} The time spent to complete this call, or null if the call is not done yet.
assigned_to {{ site.format.type.object }} Employee to which this call is assigned, or null if the call is unassigned.
customer {{ site.format.type.object }} Compact representation of the customer linked to this call, or null if no customer linked.
contact {{ site.format.type.object }} Compact representation of the contact linked to this call, or null if no contact linked.
project {{ site.format.type.object }} Compact representation of the project linked to this call, or null if no project linked.
opportunity {{ site.format.type.object }} Compact representation of the opportunity linked to this call, or null if no opportunity linked.
