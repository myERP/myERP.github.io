JSON Format

Vendors are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned when creating customers
type {{ site.format.type.integer }} yes no The type of the vendor
1: Company
2: Individual
full_name {{ site.format.type.string }} no yes The full name of this customer (automatically generated)
address {{ site.format.type.address }} no no The main address of the vendor
phone_number {{ site.format.type.string }} no no The phone number of this vendor
email {{ site.format.type.string }} no no The Email address of this vendor
website {{ site.format.type.string }} no no The URL of the vendor's website
contacts {{ site.format.type.contact }} no no An array containing the contacts of the vendor
payment_terms_id {{ site.format.type.integer}} no no The ID of the Payment terms
by default, Due on receipt will be selected.
currency_iso_code {{ site.format.type.string }} no no The ISO Code of one of your Currencies
by default, the domain currency will be selected.
user_code {{ site.format.type.string }} no no The number assigned to this vendor (i.e. SUP-002). If you don't specify a vendor number, it will be automatically generated
notes {{ site.format.type.text }} no no A note about this vendor
taxpayer_id_number {{ site.format.type.string }} no no The Taxpayer ID Number

Company only

If your vendor is a company, the JSON object will have the following keys: {% include table-header.html %}
company_name {{ site.format.type.string }} yes no The name of the company

Individual only

If your vendor is an individual, the JSON object will have the following keys: {% include table-header.html %}
first_name {{ site.format.type.string }} no no The First Name of the vendor
last_name {{ site.format.type.string }} yes no The Last Name of the vendor
middle_name {{ site.format.type.string }} no no The Middle Name of the vendor
