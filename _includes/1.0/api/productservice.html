JSON Format

Products and Services are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned when creating Transactions
type {{ site.format.type.integer }} yes no The type
1: Service
2: Product
name {{ site.format.type.string }} yes no The name. 255 characters max.
unit_id {{ site.format.type.integer }} no no The ID of the Unit
Default:
Service: Day
Product: Piece
sales_price {{ site.format.type.double }} no no The Sales Price, 0.0 by default
item_number {{ site.format.type.string }} no no A unique user-facing id (i.e. ITM-012) If you don't specify a product number, it will be automatically generated. 64 characters max.
description {{ site.format.type.string }} no no A description of the product or the service
family_id {{ site.format.type.integer }} no no IDs from Item's Families
Miscellaneous, by default
private_note {{ site.format.type.string }} no no A private note about the product or the service

Products only

{% include table-header.html %}
purchase_price {{ site.format.type.double }} no no The Purchase Price of the product. 0.0 by default
net_weight {{ site.format.type.double }} no no The Net Weight of the product. 0.0 by default
width {{ site.format.type.double }} no no The Width of the product. 0.0 by default
length {{ site.format.type.double }} no no The Length of the product. 0.0 by default
height {{ site.format.type.double }} no no The Height of the product. 0.0 by default
dimension_unit_id {{ site.format.type.integer }} no no The id of the Dimension Unit of the product. Meter by default
