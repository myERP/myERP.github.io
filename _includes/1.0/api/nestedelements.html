Address

In OneUp, addresses are used at several places. Accounts, Suppliers, Contacts... So, in the API, addresses are embeded into their own JSON format, and follow the same rules everywhere.

JSON Format

Addresses are represented as JSON objects which have the following keys:

{% include table-header.html %}
street_line1 {{ site.format.type.string }} no no First line of the address. 255 characters max.
street_line2 {{ site.format.type.string }} no no Second line of the address. 255 characters max.
street_line3 {{ site.format.type.string }} no no Third line of the address. 255 characters max.
postal_code {{ site.format.type.string }} no no The zip code of the address. 128 characters max.
city {{ site.format.type.string }} no no The city of the address. 128 characters max.
country {{ site.format.type.string }} no no The country of the address. 128 characters max.
locality {{ site.format.type.string }} no no The localisaton of the address (limited to the one from the country selected). 255 characters max.
short_locality {{ site.format.type.string }} no yes The short code for the Country. 10 characters max.
full_address {{ site.format.type.string }} no yes The full formated adress as it should appears on documents

Contact

JSON Format

Contacts are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned
first_name {{ site.format.type.string }} no no The First Name of the contact. 255 characters max.
last_name {{ site.format.type.string }} yes no The Last Name of the contact. 255 characters max.
full_name {{ site.format.type.string }} no yes The full name of this contact
cell_phone {{ site.format.type.string }} no no The Cell phone of this contact. 128 characters max.
work_phone {{ site.format.type.string }} no no The Work phone of this contact. 128 characters max.
home_phone {{ site.format.type.string }} no no The Home phone of this contact. 128 characters max.
email {{ site.format.type.string }} no no The Email address of this contact. 255 characters max.

Order Line

Order lines are product or service lines that are embeded within sales or purchases documents. They contain infos like price, quantity, description, etc...
While adding, removing or updating an order line, related values (such as total, tax amount...) on the contextual document will be updated accordingly.

JSON Format

Order lines are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned
item_id {{ site.format.type.integer }} yes yes Id of the ordered Product or Service.
description {{ site.format.type.string }} no no A description of the ordered Product or Service.
Default: The Product or Service default description
quantity {{ site.format.type.double }} no no The ordered quantity.
Default: 1.00
unit_price_wt {{ site.format.type.double }} no no Unit Price before Taxes
Default: The Product or Service default sales price
discount_rate {{ site.format.type.double }} no no Discount rate for this line.
Default: 0.00
tax_id {{ site.format.type.integer }} no no Not available for US.
Id of the Tax to apply.
is_taxable {{ site.format.type.boolean }} no no US only.
Shall the tax be applied ?
total {{ site.format.type.double }} no yes Total before taxes and discount.

Snippet

A Snippet is a generic type that encapsulates an object and aggregates common information.

JSON Format

Snippets are represented as JSON objects which have the following keys:

{% include table-header-readonly.html %}
id {{ site.format.type.integer }} The id of the encapsulated object
label {{ site.format.type.string }} The label of the encapsulated object
user_code {{ site.format.type.string }} The user_code of the encapsulated object
