Error Codes and Responses

In order to provide more informative error responses, our API attempts to return appropriate HTTP status codes for every request and return errors via a JSON error object containing error_code and error_message attributes.

myERP Code HTTP Status Code Message
003 404 The resource was not found.
005 422 The JSON request was not parsed. Please double-check the content of the request.
006 422 The JSON request is too big. Please double-check the documentation to see the current limitation.
007 500 Oops something went wrong. Please contact us so the MyERP team can investigate.
008 409 This object cannot be deleted.
009 500 The object cannot be serialized.
010 412 A mandatory field is missing in your JSON.
011 409 You are trying to edit a field that is not editable.
012 404 Data out of range.
