Address

In myERP, addresses are used at several places. Accounts, Suppliers, Contacts... So, in the API, addresses are embeded into their own JSON format, and follow the same rules everywhere.

JSON Format

Addresses are represented as JSON objects which have the following keys:

{% include table-header.html %}
label1 {{ site.format.type.string }} no no First line of the address
label2 {{ site.format.type.string }} no no Second line of the address
label3 {{ site.format.type.string }} no no Third line of the address
zip_code {{ site.format.type.string }} no no The zip code of the address
city {{ site.format.type.string }} no no The city of the address
country {{ site.format.type.string }} no no The country of the address
localisation {{ site.format.type.string }} no no The localisaton of the address (limited to the one from the country selected)
short_localisation {{ site.format.type.string }} no yes The short code for the Country
full_address {{ site.format.type.string }} no yes The full formated adress as it should appears on documents

Contact

JSON Format

Contacts are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned
first_name {{ site.format.type.string }} no no The First Name of the contact
last_name {{ site.format.type.string }} yes no The Last Name of the contact
full_name {{ site.format.type.string }} no yes The full name of this contact
cell_phone {{ site.format.type.string }} no no The Cell phone of this contact
work_phone {{ site.format.type.string }} no no The Work phone of this contact
home_phone {{ site.format.type.string }} no no The Home phone of this contact
email {{ site.format.type.string }} no no The Email address of this contact
address {{ site.format.type.address }} no no The address of this contact
notes {{ site.format.type.string }} no no A note about this contact
