Bank Entries

{% highlight bash %}Endpoint: {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.bank_entries.url}}{% endhighlight %}

This API doesn't handle PUT requests.

JSON Format

Bank entries are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned.
external_id {{ site.format.type.string }} no yes Used primilary for sync purposes.
bank_account {{ site.format.type.object }} yes yes Code of the Bank Account that entry belongs to.
posted_date {{ site.format.type.date }} yes yes The date that entry was posted.
Format: {{site.format.date}}
transaction_date {{ site.format.type.date }} yes yes The date the bank transaction was actually performed.
Format: {{site.format.date}}
amount {{ site.format.type.double }} yes yes Amount of the bank transaction. Use negative amount for debit.
Should not be 0.00
currency_iso_code {{ site.format.type.string }} no yes The ISO Code of the bank account.
description {{ site.format.type.string }} yes yes Short description.
payment_method {{ site.format.type.object }} no yes Code of the payment method.
Default: 1 (check)
provider_name {{ site.format.type.string }} no no The source of the bank entry.
Values are: NONE, YODLEE or LINXO
provider_category_id {{ site.format.type.integer }} no no The category id from the source provider (YODLEE, LINXO).
status {{ site.format.type.string }} no yes The status of the bank entry.
reconciled_documents array[{{ site.format.type.object }}] no yes The list of documents reconciled to this entry.

Available Filter Field

{% include table-header-filter.html %}
id Filter by one or multiple ids (e.g. ?id=1,2,3).
external_id Filter by one or multiple external ids (e.g. ?external_id=1,2,3).
bank_account Filter by one or multiple bank_account ids (e.g. ?bank_account=1,2,3).
posted_date Filter by posted_date.
transaction_date Filter by transaction_date.
status Filter by one or multiple status among :
- unprocessed
- unreconciled
- partially_reconciled
- to_review
- reconciled
- closed
(e.g. ?status=partially_reconciled,reconciled).

Available Sorting Options

{% include table-header-sorting.html %}
id Sorts by id field.
date Sorts by posted date field.
name Sorts by label field.
absolute_amount Sorts by the absolute value of the amount field.
