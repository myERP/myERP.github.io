h2. PUT customers/:id

p. Update a single customer or lead.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Non editable fields: id, full_name, contacts

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}}/:id {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T fieldsToUpdate.json
{% endhighlight %}

p. JSON response

The up-to-date JSON of the customer you've just modfied.

span(anchor#customers_post). ~

h2. POST customers

p. Create a single customer or lead.

h3. Resource Information

* Rate limited: Yes
* Authentication: Required
* HTTP Methods: POST
* Body Format: JSON
* Response Format: JSON
* Mandatory fields : type, status, company_name (for Company)/last_name (for Individual)

h3. Example Request

{% highlight bash %}
curl -X POST {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T newCustomer.json
{% endhighlight %}

p. JSON response

The complete JSON of the customer, as it was created by the system.

span(anchor#customers_put). ~

h2. PUT customers

p. Create and Update several customers or leads in a row.

h3. Resource Information

* Rate limited: Yes
* Batch limit: {{ site.api.batch_limit }}
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Mandatory fields for creation: type, status, company_name (for Company)/last_name (for Individual)
* Non editable fields for edition: id, full_name, contacts

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T listOfCustomers.json
{% endhighlight %}

p. JSON response

A JSON list made of up-to-date info of customers successfully edited or created, and error messages for unsuccessful ones.
