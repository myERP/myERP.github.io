Glossary

Types

Data type that can be send within JSON files.

String Any chain of characters. Carriage returns will be ignored.
String4 Any chain up to 4 characters. Carriage returns will be ignored.
String10 Any chain up to 10 characters. Carriage returns will be ignored.
String64 Any chain up to 64 characters. Carriage returns will be ignored.
String128 Any chain up to 128 characters. Carriage returns will be ignored.
String255 Any chain up to 255 characters. Carriage returns will be ignored.
Text Any chain of characters, including carriage returns, with no size limit.
Integer Any positive integer.
Long Any integer, positive or negative.
Double Any decimal number, positive or negative.
Boolean false or true. Not case sensitive.
Date Date format that should match yyyy-MM-dd. e.g. 2013-06-26.
Address In the API, Addresses got their own JSON format. Cick "here":/docs/1.0/api/address/#format for further details.
Order Line In the API, Order Lines got their own JSON format, and share it for Estimate, "Sales Orders":/docs/1.0/api/sales_order/#sales_order_format, Invoices, etc... Cick "here":/docs/1.0/api/order_line/#format for further details.
Entry Entries are the main content of any "Transaction":/docs/1.0/api/transaction/#transaction_format. They mostly define the amount of the transaction (rounded to two decimal places). They are not editable. Cick "here":/docs/1.0/api/entry/#format for further details.
