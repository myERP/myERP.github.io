h1(blue newsection#account_contact). accounts/:account_id/contacts/:id

h2(#account_contact_get). GET accounts/:account_id/contacts/:id

p. Returns all the information about the account contact. This contact should belongs to the account _account_id_.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: GET
* Response Format: JSON

h3. Example Request

{% highlight bash %}
curl -X GET {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.accounts.url}}/:account_id{{ site.resource.contacts.url}}/:id {{site.api.auth_pattern}}
{% endhighlight %}

h2(#account_contact_put). PUT accounts/:account_id/contacts/:id

p. Update a single account contact. This contact should belongs to the account _account_id_.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Non editable fields: id, customer_id

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.accounts.url}}/:account_id{{ site.resource.contacts.url}}/:id {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T fieldsToUpdate.json
{% endhighlight %}

p. JSON response

The up-to-date JSON of the account contact you've just modfied.

h2(#account_contacts_post). POST accounts/:account_id/contacts

p. Create a single account contact. This contact will belongs to the account _account_id_.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: POST
* Body Format: JSON
* Response Format: JSON
* Mandatory fields: last_name

h3. Example Request

{% highlight bash %}
curl -X POST {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.accounts.url}}/:account_id{{ site.resource.contacts.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T newAccountContact.json
{% endhighlight %}

p. JSON response

The complete JSON of the account contact, as it was created by the system.


h2(#account_contacts_put). PUT accounts/:account_id/contacts

p. Create and Update several account contacts in a row. All should or will belongs to the account _account_id_.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Mandatory fields for creation: last_name
* Non editable fields for edition: id, customer_id

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.accounts.url}}/:account_id{{ site.resource.contacts.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T listOfAccountContacts.json
{% endhighlight %}

p. JSON response

A JSON list made of up-to-date info of contacts successfully edited or created, and error messages for unsuccessful ones.
