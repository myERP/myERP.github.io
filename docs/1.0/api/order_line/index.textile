h1(blue). Order Line Format

h3. General

p. In myERP, order lines are used in several places to register an order of a product or service, where information like price, quantity, etc... are requiered. So, in the API, order lines are embeded into their own JSON format, and follow the same rules everywhere.

h3. Context

p. Order lines don't exist on their own. They exist within a context. They are _inner objects_ embeded into other data and in order to access them you have to declare the context within the URI, like _invoices/154/order_lines/257_. 

h3. Available contexts

|_. Context |_. Context URI |_. URI sample |
| "Sales Order":/docs/1.0/api/sales_order/#format | sales_orders/:sales_order_id | sales_orders/5/order_lines/17 |

span(anchor#order_line_get). ~ 

h2. GET ../order_lines/:id

p. Returns all the information about a single order line.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: GET
* Response Format: JSON

h3. Example Request

{% highlight bash %}
curl -X GET {{ site.api.base_url }}{{ site.api.v1}}/..{{ site.resource.order_lines.url}}/:id {{site.api.auth_pattern}}
{% endhighlight %}

p. JSON response sample

{% highlight bash %}
{
  "id": 847,
  "external_id": "NIN-WII-005",
  "item": 463,
  "description": "Nintendo WiiU\nBlack",
  "quantity": 5.0,
  "unit_price_wt": 299.99,
  "discount_rate": 0.0,
  "isTaxable": false,
  "total": 1499.95
}
{% endhighlight %}



span(anchor#order_line_put). ~ 

h2. PUT ../order_lines/:id

p. Update a single order line. Related values on the contextual document will be updated accordingly.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Non editable fields: id, item, total

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}/..{{ site.resource.order_lines.url}}/:id {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T fieldsToUpdate.json
{% endhighlight %}

p. JSON response

The up-to-date JSON of the order line you've just modfied.

span(anchor#order_lines_put). ~ 

h2. PUT ../order_lines

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Mandatory fields for creation: item
* Non editable fields for edition: id, item, total

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}/..{{ site.resource.order_lines.url}} {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T listOfOrderLines.json
{% endhighlight %}

p. JSON response

A JSON list made of up-to-date info of order lines successfully edited or added, and error messages for unsuccessful ones.
