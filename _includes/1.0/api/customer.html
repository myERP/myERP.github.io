JSON Format

Customers are represented as JSON objects which have the following keys:

{% include table-header.html %}
id {{ site.format.type.integer }} no yes Automatically assigned when creating customers
type {{ site.format.type.integer }} yes no The type of the customer
1: Company
2: Individual
status {{ site.format.type.integer }} no no 1: Customer (default)
2: Lead
full_name {{ site.format.type.string }} no yes The full name of this customer (automatically generated).
address {{ site.format.type.address }} no no The main address of the customer
shipping_address {{ site.format.type.address }} no no The shipping address of the customer
phone_number {{ site.format.type.string }} no no The phone number of this customer. 64 characters max.
email {{ site.format.type.string }} no no The Email address of this customer. 255 characters max.
website {{ site.format.type.string }} no no The URL of the customer's website. 255 characters max.
contacts {{ site.format.type.contact }} no no An array containing the contacts of the customer
payment_terms_id DEPRECATED {{ site.format.type.integer}} no no The ID of the Payment terms
by default, Due on receipt will be selected.
payment_terms {{ site.format.type.payment_terms}} no no The Payment terms object
by default, Due on receipt will be selected.
currency_iso_code {{ site.format.type.string }} no no The ISO Code of one of your Currencies
by default, the domain currency will be selected. 4 characters max.
sales_tax_id DEPRECATED {{ site.format.type.integer}} no no See sales_tax
sales_tax {{ site.format.type.tax}} no no A Tax object
user_code {{ site.format.type.string }} no no The number assigned to this customer (i.e. CUS-002). If you don't specify a customer number, it will be automatically generated. 64 characters max.
opt_in_to_emails DEPRECATED {{ site.format.type.boolean }} no no See opt_in_email.
opt_in_email {{ site.format.type.boolean }} no no Opt-in to Emails. Default: false
note DEPRECATED {{ site.format.type.string }} no no See notes
notes {{ site.format.type.string }} no no A note about this customer
employee_id DEPRECATED {{ site.format.type.integer }} no no See assigned_to
assigned_to Object no no The employee object. The authenticated user by default
employee_count {{ site.format.type.integer}} no no The company's employee count
price_family Object no no The price family applied to this customer
taxpayer_id {{ site.format.type.string}} no no The customer's taxpayer id
industry {{ site.format.type.string}} no no The customer's industry
incorporation_status {{ site.format.type.string}} no no The customer's incorporation status
rating {{ site.format.type.string}} no no The lead's rating

Company only

If your customer is a company, the JSON object will have the following keys: {% include table-header.html %}
company_name {{ site.format.type.string }} yes no The name of the company. 255 characters max.

Individual only

If your customer is an individual, the JSON object will have the following keys: {% include table-header.html %}
first_name {{ site.format.type.string }} no no The First Name of the customer. 255 characters max.
last_name {{ site.format.type.string }} yes no The Last Name of the customer. 255 characters max.
middle_name {{ site.format.type.string }} no no The Middle Name of the customer. 255 characters max.
