h1(blue newsection). General

p. In myERP, order lines are used in several places to register an order of a product or service, where information like price, quantity, etc... are requiered. So, in the API, order lines are embeded into their own JSON format, and follow the same rules everywhere.

h1(blue newsection). Context

p. Order lines don't exist on their own. They exist within a context. They are _inner objects_ embeded into other data and in order to access them you have to declare the context within the URI, like _invoices/:invoice_id/order_line/:id_. 

h3. Available context

|_. Context URI |_. Description |_. Context Type |_. In context URI sample |
| /sales_orders/:sales_order_id | Order lines for Sales order | "Sales Order":/docs/1.0/api/sales_order/#format | /sales_orders/:sales_order_id/order_line/:id |

h1(blue newsection#order_line). order_lines/:id

h2(#order_line_get). GET ../order_line/:id

p. Returns all the information about the sales order, including its order lines.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: GET
* Response Format: JSON

h3. Example Request

{% highlight bash %}
curl -X GET {{ site.api.base_url }}{{ site.api.v1}}/../{{ site.resource.order_lines.url}}/:id {{site.api.auth_pattern}}
{% endhighlight %}

p. JSON response sample

{% highlight bash %}
{
  "id": 847,
  "item": 463,
  "description": "Nintendo WiiU\nBlack",
  "quantity": 5.0,
  "unit_price_wt": 299.99,
  "discount_rate": 0.0,
  "isTaxable": false,
  "total": 1499.95
}
{% endhighlight %}


h2(#order_line_put). PUT ../order_lines/:id

p. Update a single sales order. Use /order_lines/:id to edit order lines.

h3. Resource Information

* Rate limited: No
* Authentication: Required
* HTTP Methods: PUT
* Body Format: JSON
* Response Format: JSON
* Non editable fields: id, user_code

h3. Example Request

{% highlight bash %}
curl -X PUT {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.order_lines.url}}/:id {{site.api.auth_pattern}} -H 'Content-Type: application/json' -T fieldsToUpdate.json
{% endhighlight %}

p. JSON response

The up-to-date JSON of the sales order you've just modfied.
