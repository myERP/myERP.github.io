Using the Access Token

Your application can use the access token to authenticate requests to OneUp API as well as transparently login the user in OneUp web application.

OneUp API

Your application must provide the access token in the header Authorization for each request to the API.

The application uses the Bearer authentication scheme to transmit the access token, as shown in the examples below:

{% highlight http %} GET {{site.api.v1}}/me HTTP/1.1 Host: {{ site.api.domain }} Authorization: Bearer xxxxxxxxxx {% endhighlight %} {% highlight bash %} curl -D- --request GET '{{site.api.protocol}}://{{ site.api.domain }}{{site.api.v1}}/me' \ --header 'Authorization: Bearer xxxxxxxxxx' {% endhighlight %}

OneUp web application

You can redirect the user to our web application and automatically log the user in, by redirecting him to the URL below and using the query parameter authentication_token: {% highlight http %} GET https://app.oneup.com{{site.web.endpoints.authorization.url}}?authentication_token=xxxxx {% endhighlight %}
