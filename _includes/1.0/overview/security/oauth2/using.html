Using the Access Token

The Application can use the access token to authenticate requests to OneUp API as well as transparently login the user in OneUp WEB Portal.

For both cases, the application must provide the token in the Authorization request header field.

The application uses the Bearer authentication scheme to transmit the access token, as shown in the examples below:

OneUp API

{% highlight http %} GET {{site.api.v1}}/xxxxxxxxxx HTTP/1.1 Host: {{ site.api.domain }} Authorization: Bearer xxxxxxxxxx {% endhighlight %} {% highlight bash %} curl -D- --request GET '{{site.api.protocol}}://{{ site.api.domain }}{{site.api.v1}}/xxxxxxxxxx' \ --header 'Authorization: Bearer xxxxxxxxxx' {% endhighlight %}

OneUp WEB Portal

{% highlight http %} POST {{site.web.endpoints.authorization.url}}/ HTTP/1.1 Host: {{ site.web.domain }} Authorization: Bearer xxxxxxxxxx {% endhighlight %} {% highlight bash %} curl -D- --request POST '{{site.web.protocol}}://{{ site.web.domain }}{{site.web.endpoints.authorization.url}}/' \ --header 'Authorization: Bearer xxxxxxxxxx' {% endhighlight %}
