h1(blue). Format

h3. Address

p. In myERP, addresses are used at several places. Accounts, Suppliers, Contacts... So, in the API, addresses are embeded into their own JSON format, and follow the same rules everywhere.


|_. {{ site.format.header.field_name }} |_. {{ site.format.header.description }} |_. {{ site.format.header.type }} |_. {{ site.format.header.values }} |_. {{ site.format.header.default }} |_. {{ site.format.header.non_editable }} |
| label1 | First line of the address | {{ site.format.type.string }} |  | Empty String |  |
| label2 | Second line of the address | {{ site.format.type.string }} |  | Empty String |  |
| label3 | Third line of the address | {{ site.format.type.string }} |  | Empty String |  |
| zip_code | The zipcode code | {{ site.format.type.string }} |  | Empty String |  |
| city | The city name | {{ site.format.type.string }} |  | Empty String |  |
| country | The country name | {{ site.format.type.string }} |  | Empty String |  |
| localisation | The Country/County | {{ site.format.type.string }} | Limited to the one from your country. | Other |  |
| short_localisation | The short code for the Country | {{ site.format.type.string }} | {{ site.format.read_only }} | {{ site.format.non_available }} |=. {{ site.format.non_editable }} |
| full_address | The full formated adress 
as it should appears on documents | {{ site.format.type.string }} | {{ site.format.read_only }} | {{ site.format.non_available }} |=. {{ site.format.non_editable }} |

p. JSON sample

{% highlight bash %}
{
  "label1": "567  Sutter St.",
  "label2": "",
  "label3": "",
  "zip_code": "94102",
  "city": "San Francisco",
  "country": "United States",
  "localisation": "California",
  "short_localisation": "CA",
  "full_address": "567  Sutter St. San Francisco CA 94102 United States"
}
{% endhighlight %}
