Bulk Deletion

If you make a lot of API requests in a short amount of time, you may bump into the OneUp rate limit for requests. When you reach the limit, OneUp stops processing any more requests until a certain amount of time has passed.

We provide you a bulk deletion method for each resource. You can delete up to {{site.api.bulk_deletion_limit}} elements at the time by passing a comma-separated list of ids in the query string. The ids parameter accepts only integers separated by commas. If this parameter is missing or contains any wrong characters, you will get a 400.

The API returns a list of errors and a list of successes. The bulk deletion API does not stop when an element cannot be deleted, this element is added to the errors list with the explanation. The successes list contains all the elements deleted.

Example

{% highlight bash %} CURL -X DELETE {{ site.api.base_url }}{{ site.api.v1}}{{ site.resource.customers.url}}?ids=12,42,1337 {{site.api.auth_pattern}} {% endhighlight %}
